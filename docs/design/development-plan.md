## 4. リスク管理

### 4.1 回答の正確性 (ハルシネーション)
-   **リスク**: 古いマニュアルと新しいUpdate資料が競合し、誤った回答をする。
-   **対策**:
    -   `metadata` に `date` を付与し、新しい日付のドキュメントを優先してコンテキストに含めるロジックを実装する。
    -   システムプロンプトで「不確実な情報は回答せず、担当者への問い合わせを促す」よう指示を徹底する。

### 4.2 音声認識の精度
-   **リスク**: 施設内の雑音環境や専門用語により、誤認識が発生する。
-   **対策**:
    -   音声入力後、即座に送信せず、一度テキストエリアに表示してユーザーが編集・確認できるUIにする。
    -   誤認識しやすい単語（製品名など）は、クイック質問ボタンでカバーする。

### 4.3 コスト超過
-   **リスク**: チャット利用急増によるAPIコストの増大。
-   **対策**:
    -   OpenAI APIのレートリミットを設定する。
    -   Supabase等の無料枠監視アラートを設定する。
    -   `gpt-4o-mini` を利用し、コストパフォーマンスを維持する。
