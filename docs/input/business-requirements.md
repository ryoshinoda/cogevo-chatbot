# ビジネス要件

## 1. 課題定義
### 解決する課題
- **問い合わせ対応のボトルネック**: 専門的な知識が必要な問い合わせがメールや電話で寄せられ、社内の一部エキスパートに負荷が集中している。
- **回答待ち時間の発生**: エスカレーションにより回答まで時間がかかり、ユーザー（施設スタッフ）の業務が止まることがある。

### 課題の深刻度
- **高**: エキスパートのリソースが圧迫され、本来の業務（開発や企画など）に支障が出ている可能性がある。

### 課題が発生する状況
- 施設スタッフがタブレットでCogEvoを利用中、操作方法やトラブルシューティングで不明点が出た際。

## 2. 市場環境
### 既存の代替手段
- **電話・メール窓口**: 現在の主な問い合わせ手段。
- **マニュアル・FAQ**: 存在するが、検索性が低い、あるいは参照されていない可能性がある。

### 既存手段の課題
- 人力対応のため、夜間・休日の対応が難しい。
- 1対1の対応となるため、件数が増えるとスケールしない。

## 3. 解決策
### アプローチ
- Gemini APIを活用したAIチャットボットを導入し、一次対応を自動化する。
- 既存のPDFマニュアル、FAQリスト、HP情報をナレッジベースとして活用する。

### 提供価値
- **即時解決**: 待たずにその場で答えが得られる。
- **専門用語の翻訳**: エンジニアの知識を持ちつつ、現場スタッフにわかりやすい言葉で説明する。

## 4. ビジネスモデル
### 位置づけ
- CogEvoサービスの付帯サポート機能（追加課金なし）。
- 顧客満足度の向上と、社内コスト（対応工数）の削減が主なリターン。

## 5. リスクと対策
### ハルシネーション（誤回答）
- **リスク**: AIが誤った操作方法を案内してしまう。
- **対策**:
  - 会話開始時に「AIのため誤りがある可能性があります」と免責事項を明記する。
  - 回答に自信がない場合や解決しない場合は、問い合わせフォームへ誘導する。
  - 重要な意思決定（契約関連など）は有人対応へ流すようプロンプトで制御する。

### 情報セキュリティ
- **リスク**: 社外秘情報が誤って漏洩する、あるいは不特定多数に利用される。
- **対策**:
  - 一般公開せず、共通パスワードによる簡易認証を設ける。
  - 学習データには「社外に出しても良い情報（マニュアル等）」のみを使用する。
